# Visualizer

Transform the storyboard's visual descriptions into generation prompts optimized for different AI models.

## Input Reference

Before writing prompts, **the following files must be read first**:
- `storyboard.yaml` — Shot composition, movement, rhythm, and other technical parameters
- `screenplay.yaml` — **Novel-level visual descriptions** (200-600 word visual_description per scene), including character appearance/clothing, object details, spatial relationships, color and lighting, time changes, etc. This is the core source material for prompts; specific visual details from the descriptions should be directly translated into prompt keywords. Also includes character visual profiles (characters.visual_definition) and global visual tone (visual_tone).

## Prompt Structure (General)

```
[subject description], [environment/background], [lighting], [color tone], [camera/composition], [style modifiers], [quality modifiers]
```

## Seedream Image Prompt Optimization

Seedream (doubao-seedream) is Doubao's text-to-image model:

- Supports both Chinese and English prompts; English is more stable
- Keywords separated by commas
- Important elements go first
- Avoid negation words (don't say "no xxx")

**Recommended structure:**
```
[subject], [action/pose], [background/environment], [lighting], [camera angle],
[style: cinematic/anime/photorealistic], [quality: masterpiece, best quality, 8k]
```

## Seedance Video Prompt Optimization

Seedance (doubao-seedance) is Jimeng's text/image-to-video model:

- Describe motion direction and speed
- Describe camera movement
- Keep it concise with a clear focus
- A first-frame image significantly improves consistency
- **Video opening and ending states must connect with adjacent shots**

**Recommended structure:**
```
[scene description], [motion: what moves and how], [camera movement],
[mood/atmosphere], [style]
```

### Shot Continuity Prompt Techniques

Video prompts should include start/end state descriptions to ensure smooth editing between adjacent shots:

```
The video begins with [opening_state from storyboard],
[main action and camera movement],
ending with [closing_state from storyboard]
```

Example:
```
The video begins with a wide shot of the ancient merchant standing
at the edge of a bamboo forest at dawn. He slowly turns and walks
into the forest, camera tracking behind him. The video ends with
his silhouette disappearing into the misty bamboo.
```

**Key Points:**
- Explicitly describe the video's starting and ending visuals in the prompt
- Ensure visual continuity with the previous/next shot
- Motion direction must be consistent with adjacent shots

## Minimax Video Prompts

Minimax Video-01:
- Supports Chinese prompts
- Excels at character movement and scene transitions
- The more detailed the prompt, the better

## Sora 2 Video Prompts

OpenAI Sora:
- English only
- Natural language descriptions, written like prose
- No keyword-style prompts needed
- Can describe complex scenes

## Consistency Strategy

### Character Anchors
Define fixed descriptions for recurring characters/objects:

```json
{
  "character_anchors": {
    "hand": "young woman's hand, slender fingers, natural short nails, fair skin",
    "bottle": "transparent glass bottle with yellow lemon sparkling water, condensation droplets"
  }
}
```

### Reference Image Driven (Recommended)

A more effective consistency method than pure text anchors. Specify `reference_images` in each shot of prompts.json;
the `generate_image` tool will automatically use `image_to_image` mode, significantly improving character and scene consistency.

```json
{
  "image_prompt": {
    "prompt": "...",
    "reference_images": ["merchant", "bamboo_forest"],
    "aspect_ratio": "9:16"
  }
}
```

Reference images need to be generated first via the `generate_reference` tool and saved under `assets/references/`.
See designer/SKILL.md for details.

### Style Anchor (style_anchor)

A **detailed style description** (50-100 words) generated by the model during the design phase based on the project's theme/mood/era, globally appended to every prompt.
The style_anchor should cover: render style, color system, lighting, texture, reference aesthetics, exclusions, and other dimensions. See designer/SKILL.md for details.

```json
{
  "style_anchor": "cinematic photorealistic, film grain, 2K resolution, golden hour natural lighting, volumetric god rays, shallow depth of field, warm amber and teal color grading, high contrast with rich shadows, epic scale compositions, Terrence Malick and Roger Deakins inspired cinematography, NOT cartoon, NOT 3D render, NOT anime"
}
```

**Every shot's prompt must fully include the style_anchor without omission or simplification.**

## Motion Realism Rules

When describing object motion in video prompts, physical realism must be followed:

1. **Objects don't move by themselves** — Every motion must have an agent. ❌ "The shell moves to the ear" → ✅ "The boy lifts the shell to his ear with his left hand"
2. **Describe hand/body actions** — Don't describe objects "drifting," "floating," or "appearing"; describe how the character's hands pick up, set down, or hand over
3. **One motion per shot** — Don't describe multiple complex actions in a single 3-5 second shot
4. **Describe camera movement and subject movement separately** — "Camera slowly pushes in while the boy turns his head left" rather than mixing them together
5. **Forbidden vocabulary** — appears, floats, emerges, materializes, hovers — these all imply physically impossible motion
6. **Replacement vocabulary** — picks up, lifts, holds, places, reaches for, sets down, carries

## Prompt Optimization Principles

1. **Specific > Abstract** — "soft warm sunlight at 5pm" not "nice lighting"
2. **Front > Back** — Put the most important elements at the beginning of the prompt
3. **Less is more** — Don't pile on too many keywords; focus each prompt on 3-5 core elements
4. **Avoid conflicts** — Don't simultaneously describe "dark moody" and "bright cheerful"
5. **Test and iterate** — Generate one image to check the effect, then adjust the prompt
